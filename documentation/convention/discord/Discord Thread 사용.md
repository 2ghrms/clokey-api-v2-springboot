# Discord Thread Usage Guide (Clokey)

> 최종 수정 일자 : 8월 28일
>
> 최종 수정자 : 이호근

### 목적

Clokey 디스코드 서버는 파트별 협업과 전체 프로젝트 진행 상황 공유를 위해 사용함. 스레드를 적극적으로 활용해 대화를 주제별로 정리하고, 모든 파트가 업무 현황을 인식할 수 있도록 하는 것이 목표임.

### 채널 구조

디자인 / iOS / SpringBoot / AI·ML / 운영홍보 파트 카테고리 존재

### 각 파트 카테고리 내 채널 구성

업무방 (텍스트): 해당 파트의 업무 진행, 공유, 내부 질문

질문방 (텍스트): 다른 파트에 질문할 때 사용

회의실 (음성): 회의 및 실시간 논의

### 기본 원칙

업무 공유는 반드시 디스코드에서 진행
모든 파트는 업무 현황을 인식할 수 있어야 하므로, 모든 업무 관련 내용은 디스코드에 기록해야 함.

카카오톡은 업무 채널이 아님
카톡은 단순히 "확인 부탁"과 같은 빠른 전달, 주제가 없는 간단한 공유용으로만 사용함.

모든 업무 논의는 스레드 필수
채널에 글을 남기면 반드시 스레드를 열어 후속 논의를 이어가야 함.

### 스레드 생성 규칙

다른 파트에 질문할 경우 → 질문방에서 스레드 생성

본인 파트의 업무 공유 및 내부 논의 → 업무방에서 스레드 생성

스레드 이름은 주제 키워드 중심으로 간결하게 작성하되, 너무 일반적이지 않게 작성할 것

예시: 로그인 버그 리포트 공유, UI 프로필 수정 방식 논의, 데이터셋 전처리 방식 관련 질문

### 스레드 사용 규칙

스레드는 해당 주제와 직접적으로 관련된 대화만 이어감

잡담, 소셜 대화는 스레드 대신 일반 채널이나 카톡 활용

논의 형식이라면, 반드시 결론/산출물 요약을을 업무 채널에 공유하여 팀원이 팔로우할 수 있도록 할 것

완료된 스레드는 종료 표시하거나 아카이브 처리

### 알림 관리

@mention은 꼭 필요한 경우에만 사용

특정인 언급 대신 역할(role) 태그나 파트별 태그 활용

### 예시 시나리오

SpringBoot 파트에서 AI 파트에 질문
→ AI/ML 질문방에서 스레드 생성 → 답변 및 논의 진행 → 최종 결론 요약하여 팀원에게 공유

디자인 파트 내부 UI 검토
→ 디자인 업무방에서 스레드 생성 → 논의 후 결과 공유

운영홍보 파트 공지 디자인 피드백 요청
→ 운영홍보 질문방에서 스레드 생성 → 디자인/IOS/SpringBoot 피드백 수집 → 결과 정리

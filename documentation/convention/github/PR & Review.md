# 📌 PR

자동 관리

- PR Template은 ```.github에 있는 것을 사용하시면 됩니다.```
- ⭐현재 Acitve한 유저들은 ```.github/CODEOWNER```에 기재되어 있고 git action에 의해서 자동으로 PR Reviewer로 태그됩니다.
- 따라서, 새로운 인원이 추가되면 이 부분 업데이트가 필요합니다.

수동 관리

- 꼭 본인을 ```assignees```로 지정하고  ```Label```을 지정해 주세요.

PR 내용 작성

- PR의 내용은 최대한 꼼꼼하게 템플릿에 맞게 작성합니다. 
- 다만, 필요없는 탭은 비워두지 말고 그냥 지워서 PR을 올려도 좋습니다.

# 📌 Review

- Review는 최대한 꼼꼼히 부탁드립니다.
- ⭐PR에 보이지 않는 부분도 코드의 줄에 작성을 해주세요. (댓글을 달기 불편합니다)
- ⭐ 수정 사항을 ```요청한 사람```이 해결되었다고 판단하는 경우 Resolve를 해주시면 됩니다. (리뷰를 받는 사람이 닫지 말아주세요.).

### ⭐⭐⭐ PR 수정 커밋

- PR 수정 커밋은 최대한 안건(댓글) 하나당 하는 것이 좋습니다. 
- 하지만, 수정 사항이 많아질 경우 위의 방법 적용이 제한될 수 있습니다. 이런 경우 그래도 묶음 단위로 커밋을 부탁드립니다.
- 그렇지 않으면 리뷰어가 브랜치를 받아서 하나씩 확인해야해요!!!
